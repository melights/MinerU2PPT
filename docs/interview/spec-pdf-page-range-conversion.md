# Feature Brief: PDF Page Range Conversion

## 1. Overview
为 MinerU2PPT 增加“按页码范围转换”的能力：当输入文件为 PDF 时，用户可指定页码范围，仅转换所选页面到 PPT；当输入为非 PDF（如单图片）时，页码范围参数被忽略并按现有流程继续执行。该功能目标是减少不必要的页面处理，提高批量/长文档转换的可控性。

## 2. Goals
- 支持用户在 **CLI 和 GUI** 中指定 PDF 页码范围进行部分转换。
- 支持混合语法：**单页 + 连续区间 + 逗号分隔**（例如 `1,3,5-8`）。
- 使用 **1-based 严格校验**：非法格式或越界输入应明确报错并停止该任务。
- 对非 PDF 输入，页码范围参数 **不报错、被忽略**，保持现有行为不变。

## 3. User Stories

- **US-001: 仅转换 PDF 的指定页面**
  - **As a** CLI/GUI 用户, **I want to** 指定页码范围, **so that I can** 只转换目标页，避免全量转换。
  - **Acceptance Criteria:**
    - [ ] 输入 `1,3,5-8` 时，只输出对应页内容到 PPT。
    - [ ] 未提供页码范围时，行为与当前版本一致（全页转换）。
    - [ ] 输出页顺序与 PDF 原始页顺序一致。

- **US-002: 非法页码输入可被明确拦截**
  - **As a** CLI/GUI 用户, **I want to** 在输入错误时收到清晰提示, **so that I can** 快速修正参数。
  - **Acceptance Criteria:**
    - [ ] 非法格式（如 `1--3`、`a,b`）会报错并给出可理解提示。
    - [ ] 越界页码（如总页数 10 但输入 11）会报错并停止该任务。
    - [ ] 空范围或反向区间（如 `8-5`）会报错并停止该任务。

- **US-003: 非 PDF 输入兼容现有流程**
  - **As a** 图片输入用户, **I want to** 即使传入页码范围也不被阻断, **so that I can** 保持与现有用法兼容。
  - **Acceptance Criteria:**
    - [ ] 输入为非 PDF 时，页码范围参数被忽略。
    - [ ] 任务正常完成，不因页码参数失败。

## 4. Functional Requirements
- **FR-1**: 系统必须在 CLI 提供可选页码范围参数（适用于 PDF 输入）。
- **FR-2**: 系统必须在 GUI 提供页码范围输入能力（适用于 PDF 输入）。
- **FR-3**: 系统必须支持页码表达式语法：`N`、`N-M`、以及它们的逗号组合（如 `1,3,5-8`）。
- **FR-4**: 页码规则必须为 **1-based**，且采用**严格校验**（格式非法、越界、反向区间均报错）。
- **FR-5**: 对合法页码范围，系统必须仅转换选中页，并保持输出页顺序与源 PDF 页序一致。
- **FR-6**: 对非 PDF输入，系统必须忽略页码范围参数并继续现有转换流程。
- **FR-7**: 错误提示必须包含“错误原因 + 合法示例”（例如 `1,3,5-8`），便于用户修复输入。
- **FR-8**: 未设置页码范围时，系统行为必须与当前版本一致（无功能回归）。

## 5. Non-Goals (Out of Scope)
- 不支持奇偶页、按关键词、按章节标题等高级筛选。
- 不支持跨文件统一页码映射（仅针对单个 PDF 的页码）。
- 不包含转换后页面重排、删除、合并等 PPT 编辑功能。
- 不改变现有 OCR/合并策略本身（仅控制“处理哪些页”）。

## 6. Success Metrics
- 合法页码范围场景下，**100%** 输出页集合与预期页集合一致。
- 非法输入场景下，**100%** 触发明确错误并阻止该任务继续。
- 非 PDF + 页码参数场景下，**100%** 任务可成功完成（参数被忽略）。
- 未使用页码范围的回归场景中，现有核心转换测试通过率保持 **100%**（与当前基线一致）。

## 7. Open Questions
1. GUI 中页码范围输入位置是否需在单任务和批任务都支持（当前建议：都支持）。
2. 非 PDF 忽略页码参数时，是否需要显示轻量提示（如状态栏提示“已忽略”）。
